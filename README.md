# 明日方舟素材均衡规划器

这个项目的出发点很简单。

我后来发现，明日方舟里并不缺“为了某个指定材料该去刷哪关”的计算器，但几乎没有人单独做一个“我现在整体最缺什么材料”的工具。

这就是这个网页的初衷。

它不是先让你选目标材料，再反推去哪刷；它是先把你仓库里的素材统一折算到蓝材口径，然后直接告诉你现在整体最缺哪些蓝色素材，方便你先看仓库失衡点，再决定后面怎么刷。

它更适合这类玩家：

- 已经有一定库存，不是刚入坑的新号
- 不想每次都先指定一个目标材料
- 更关心整个仓库是不是失衡，而不只是单次刷图收益
- 想先知道“最缺什么”，再决定接下来补什么

## 网页能做什么

- 导入库存 JSON
- 默认使用内置权重
- 重新导入新的权重 JSON 覆盖默认权重
- 直接在网页里修改素材数量
- 计算当前最缺的蓝色素材
- 展开查看完整蓝材排序
- 导出当前库存 JSON

## 本地打开

先确保电脑已经安装：

- Node.js 18+
- npm

### macOS / Linux

```bash
./start.sh
```

### Windows CMD

```cmd
start.bat
```

### Windows PowerShell

```powershell
.\start.ps1
```

第一次启动时，脚本会自动安装前端依赖，然后打开本地网页。

如果你想手动启动：

```bash
cd frontend
npm ci
npm run start
```

## 怎么用

1. 打开网页。
2. 点击 `导入库存`，选择你的库存 JSON。
3. 如果你有更新的权重文件，再点击 `导入权重` 覆盖默认权重。
4. 直接在页面里改素材数量。
5. 点击 `计算规划` 查看当前最缺素材。
6. 点击 `下载库存` 导出当前库存。

## 库存文件格式

支持企鹅物流规划器使用的库存 JSON 格式。

如果你已经有现成库存文件，直接导入即可。

## 权重文件

网页默认已经内置一份权重。

如果你想更新权重，可以重新导入新的权重 JSON。

权重下载地址：

- https://ark.yituliu.cn/material/value

## GitHub Pages 部署

仓库已经带了 GitHub Pages 工作流。

你只需要：

1. 把仓库推到 GitHub。
2. 进入仓库 `Settings`。
3. 打开 `Pages`。
4. Source 选择 `GitHub Actions`。
5. 推送到 `main` 分支后，会自动部署。

## 说明

- 这是纯静态网页版本
- 页面保留 Liquid Glass 视觉效果
- 只关注素材，不处理芯片、经验书、龙门币等其他物品
- 所有品质素材都会折算到蓝材口径
